## Сервис коротких ссылок + QR на Yii2

Проект реализует сервис сокращения ссылок с генерацией QR‑кода и простой админ‑панелью статистики, построенный на стеке **Yii2 + MySQL + jQuery + Bootstrap**, без использования сторонних веб‑API.

---

### 1. Стек и требования

- **PHP**: версия 7.4+  
- **MySQL**: любая актуальная версия (InnoDB)  
- **Web‑сервер/CLI**: встроенный PHP‑сервер или Apache/Nginx  
- **Composer**: для установки PHP‑зависимостей  
- **Технологии проекта**:
  - Yii2 (basic template)
  - MySQL
  - jQuery (из комплекта Yii2)
  - Bootstrap 5 (через `yiisoft/yii2-bootstrap5`)
  - Локальная библиотека `endroid/qr-code` для генерации QR (без сторонних API)

---

### 2. Установка зависимостей

1. Перейдите в корень проекта:

```bash
cd /home/alexander/project_genomed
```

2. Установите зависимости Composer:

```bash
composer install
```

Если зависимости уже установлены, этот шаг можно пропустить.

---

### 3. Настройка подключения к базе данных

1. Создайте базу данных в MySQL, например:

```sql
CREATE DATABASE yii2basic CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. Откройте файл `config/db.php` и укажите реальные параметры подключения:

```php
return [
    'class' => 'yii\db\Connection',
    'dsn' => 'mysql:host=localhost;dbname=yii2basic',
    'username' => 'root',
    'password' => '',
    'charset' => 'utf8',
];
```

3. Убедитесь, что пользователь БД имеет права на создание таблиц/запись.

---

### 4. Применение миграций

Миграции создают необходимые таблицы для коротких ссылок и логов переходов.

Выполните в корне проекта:

```bash
cd /home/alexander/project_genomed
php yii migrate --interactive=0
```

Будут созданы таблицы:

- `short_url` — основная информация о коротких ссылках:
  - `id` — первичный ключ  
  - `code` — уникальный короткий код  
  - `original_url` — исходный URL  
  - `hit_count` — счётчик переходов  
  - `created_at`, `updated_at` — временные метки

- `short_url_hit` — логи переходов:
  - `id` — первичный ключ  
  - `short_url_id` — ссылка на `short_url.id`  
  - `ip` — внешний IP, с которого был переход  
  - `created_at` — дата и время перехода

### 6. Основной функционал сервиса

**Главная страница (`SiteController::actionIndex`)**

- Показывает форму:
  - Одно поле `Input` для ввода любого URL.
  - Кнопка **«OK»**, отправляющая запрос через **Ajax** (jQuery) на `/site/shorten`.
- Ответ от сервера обрабатывается без перезагрузки страницы:
  - При ошибке — вывод текста ошибки под формой (например «Данный URL не доступен»).
  - При успехе — отображается:
    - Сгенерированный QR‑код.
    - Короткая ссылка рядом с QR‑кодом.

**Валидация и проверка URL (`SiteController::actionShorten`)**

При нажатии **«OK»** происходит:

- Проверка валидности URL:
  - Обязателен корректный формат.
  - Разрешены только схемы `http` и `https`.
- Проверка доступности ресурса:
  - Через `cURL` отправляется запрос `HEAD`/`NOBODY`.
  - Если ресурс недоступен или возвращает некорректный код/ошибку — возвращается сообщение  
    **«Данный URL не доступен»**.
- Если валидация и доступность пройдены:
  - Генерируется уникальный короткий код.
  - В таблицу `short_url` сохраняется исходный URL и код.
  - Формируется:
    - Абсолютная короткая ссылка вида `http://host/s/XXXXXX`.
    - Внутренний URL для получения QR‑кода.
  - Всё это возвращается в формате JSON.

**Генерация QR‑кода (`SiteController::actionQr`)**

- На вход принимает короткий код.
- Генерирует QR‑картинку (PNG) для ссылки вида `http://host/s/XXXXXX`.
- Используется локальная библиотека `endroid/qr-code`:
  - Кодируется ссылка.
  - Настраивается размер, отступы и уровень коррекции ошибок.
- Контроллер отдаёт бинарный PNG‑контент напрямую в браузер.

**Переход по короткой ссылке (`RedirectController::actionGo`)**

- Маршрутизация реализована через `urlManager` в `config/web.php`:
  - Правило `s/<code:[A-Za-z0-9]+>` направляет запросы `/s/XXXXXX` в `redirect/go`.
- При обращении к `http://host/s/XXXXXX`:
  - По коду находится запись в `short_url`.
  - Если не найдена — возвращается 404.
  - Если найдена:
    - Увеличивается счётчик `hit_count`.
    - В таблицу `short_url_hit` записывается:
      - ID короткой ссылки.
      - IP пользователя.
      - Время перехода.
    - Пользователь перенаправляется (`HTTP redirect`) на исходный URL.

### 7. Админ‑панель и статистика

Админ‑функционал реализован в `AdminController` и представлениях в `views/admin`.

**Список всех коротких ссылок**

- URL:
  - С pretty‑URL: `http://localhost:8080/admin/links`
  - Без pretty‑URL: `http://localhost:8080/index.php?r=admin/links`
- Таблица (`GridView`) содержит:
  - ID и короткий код.
  - Кликая короткая ссылка (ведёт на `redirect/go`).
  - Оригинальный URL.
  - Счётчик переходов (`hit_count`).
  - Даты создания и обновления.
  - Кнопку **«Логи»** для перехода к детальной статистике по ссылке.

**Логи переходов по конкретной ссылке**

- URL:
  - С pretty‑URL: `http://localhost:8080/admin/logs?id=ID`
  - Без pretty‑URL: `http://localhost:8080/index.php?r=admin/logs&id=ID`
- Отображает:
  - Короткую ссылку и исходный URL.
  - Общее количество переходов.
  - Таблицу логов (`short_url_hit`):
    - ID записи.
    - IP‑адрес.
    - Дата/время перехода.

1. Установить PHP, MySQL, Composer.

   - `composer install`

   - `php yii migrate --interactive=0`

   - `cd web && php -S localhost:8080 -t .`
5. Открыть:
   - Главная страница сервиса: `http://localhost:8080`
   - Админ‑список ссылок: `http://localhost:8080/admin/links`
